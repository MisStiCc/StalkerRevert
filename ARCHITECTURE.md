# Архитектурные решения для "Сталкер наоборот"

## Общая архитектура

Проект представляет собой стратегию в реальном времени на движке Godot 4, где игрок управляет Зоной, защищающейся от сталкеров.

## Основные компоненты

### ZoneController
#### Переменные:
- energy: float - уровень энергии Зоны
- biomass: float - уровень биомассы Зоны
- max_energy: float - максимальный уровень энергии
- max_biomass: float - максимальный уровень биомассы
- emission_cooldown: float - время до следующего выброса
- emission_duration: float - длительность выброса
- anomalies: Array - список текущих аномалий
- mutants: Array - список текущих мутантов
- artifacts: Array - список текущих артефактов
- stalkers: Array - список текущих сталкеров
- territory_radius: float - радиус территории Зоны

#### Функции:
- init() -> void - инициализация контроллера
- _process(delta: float) -> void - основной цикл обновления
- add_energy(amount: float) -> void - добавление энергии
- spend_energy(amount: float) -> bool - расход энергии
- add_biomass(amount: float) -> void - добавление биомассы
- spend_biomass(amount: float) -> bool - расход биомассы
- create_anomaly(type: String, position: Vector2) -> Anomaly - создание аномалии
- spawn_mutant(type: String, position: Vector2) -> Mutant - призыв мутанта
- do_emission() -> void - выполнение выброса
- expand_territory(radius_increase: float) -> void - расширение территории
- generate_artifact(position: Vector2) -> Artifact - генерация артефакта
- update_stalker_status(stalker: Stalker) -> void - обновление статуса сталкера
- get_resource_efficiency() -> float - эффективность использования ресурсов

#### Сигналы:
- energy_changed(new_energy: float)
- biomass_changed(new_biomass: float)
- anomaly_created(anomaly: Anomaly)
- mutant_spawned(mutant: Mutant)
- emission_started()
- emission_ended()
- territory_expanded(new_radius: float)
- artifact_generated(artifact: Artifact)
- stalker_entered_zone(stalker: Stalker)
- stalker_left_zone(stalker: Stalker)
- stalker_died(stalker: Stalker)

#### Взаимодействие с другими объектами:
- Anomaly: создает и управляет аномалиями, получает уведомления о их состоянии
- Mutant: создает и управляет мутантами, получает уведомления о их состоянии
- Stalker: отслеживает позиции и действия сталкеров, применяет эффекты
- Artifact: создает и отслеживает артефакты
- UI: предоставляет данные для интерфейса

### Anomaly System
#### Базовый класс Anomaly:
- Параметры: позиция, радиус действия, длительность, периодичность эффекта
- Методы: apply_effect(target), update(delta), is_active()
- Сигналы: anomaly_activated, anomaly_deactivated, effect_applied

#### Типы аномалий:
- Heat Anomaly ("Жарка"):
  * Параметры: урон, радиус, частота импульсов
  * Эффект: постепенный урон огнем
  * Цвет: красный
  
- Electric Anomaly ("Электра"):
  * Параметры: урон, радиус, шанс оглушения, длительность оглушения
  * Эффект: урон электричеством, кратковременное оглушение
  * Цвет: синий
  
- Acid Anomaly ("Кислота"):
  * Параметры: урон, радиус, снижение брони, длительность эффекта
  * Эффект: урон кислотой, временное снижение защиты
  * Цвет: зеленый
  
- Gravitational Anomaly ("Грави"):
  * Параметры: радиус, сила притяжения, урон от сдавливания
  * Эффект: притягивание объектов к центру, урон при столкновении
  * Цвет: фиолетовый

#### Взаимодействие с ZoneController:
- ZoneController создает аномалии через метод create_anomaly()
- Аномалии отправляют сигналы при активации/деактивации
- ZoneController отслеживает все аномалии в массиве anomalies
- При удалении аномалии ZoneController получает уведомление

### Stalker System
#### Базовый класс Stalker:
- Параметры: здоровье, максимальное здоровье, броня, урон, скорость, имя
- Методы: take_damage(amount, type), move_to(target), collect_artifact(artifact), avoid_anomaly(anomaly), update(delta)
- Сигналы: stalker_damaged, stalker_died, artifact_collected, entered_zone, left_zone

#### Типы сталкеров:
- Novice Stalker ("Новичок"):
  * Параметры: здоровье 50, скорость 80, урон 10, броня 0
  * Поведение: простое движение к артефактам, минимальная реакция на аномалии
  * Стоимость для Зоны: низкая (легко уничтожить)
  
- Veteran Stalker ("Ветеран"):
  * Параметры: здоровье 100, скорость 100, урон 20, броня 10
  * Поведение: средняя осторожность при обходе аномалий, групповое поведение
  * Стоимость для Зоны: средняя
  
- Legend Stalker ("Легенда"):
  * Параметры: здоровье 200, скорость 120, урон 35, броня 25
  * Поведение: высокая осторожность, уклонение от мутантов, координация с другими сталкерами
  * Стоимость для Зоны: высокая (трудно уничтожить)

#### Поведение:
- Поиск артефактов: сталкеры используют систему навигации для поиска и сбора артефактов
- Обход аномалий: разные типы сталкеров по-разному реагируют на аномалии в зависимости от их характеристик
- Группировки: опытные сталкеры могут действовать в группах, координируя свои действия

#### Взаимодействие с ZoneController:
- ZoneController отслеживает всех сталкеров в массиве stalkers
- При входе в зону сталкер отправляет сигнал
- При смерти сталкер отправляет сигнал и вознаграждение для Зоны

### Artifact System
#### Базовый класс Artifact:
- Параметры: позиция, тип, ценность, эффект (опционально)
- Методы: collect(stalker), apply_effect(target), update(delta)
- Сигналы: collected_by_stalker(stalker), picked_up

#### Типы артефактов:
- Common Artifact ("Коллекционный"):
  * Параметры: ценность 10, эффект отсутствует
  * Специфика: простой для получения, базовая цель для новичков
  
- Rare Artifact ("Редкий"):
  * Параметры: ценность 25, эффект: временно увеличивает урон сталкера
  * Специфика: более ценный, дает бонусы при сборе
  
- Unique Artifact ("Уникальный"):
  * Параметры: ценность 50, эффект: устойчивость к определенным аномалиям
  * Специфика: самый ценный, дает значительные преимущества

#### Механизмы:
- Сбор сталкерами: сталкеры могут собирать артефакты, подойдя к ним
- Создание Зоной: ZoneController может генерировать артефакты в случайных местах
- Защита: артефакты могут быть защищены аномалиями или мутантами

#### Взаимодействие с ZoneController:
- ZoneController отслеживает все артефакты в массиве artifacts
- При сборе артефакта ZoneController получает уведомление
- ZoneController может создавать новые артефакты для привлечения сталкеров

### Mutant System
- Базовый класс мутанта
- Типы мутантов (Собаки, Снорки, Контролёры)
- Поведение (патруль, атака на сталкеров)

## Система ресурсов

### Энергия
- Используется для создания аномалий
- Накапливается со временем или через специальные объекты

### Биомасса
- Используется для призыва мутантов
- Накапливается через уничтожение сталкеров или другие механизмы

## UI система
- Панель ресурсов
- Кнопки создания аномалий
- Индикаторы состояния
- Карта уровня
