# Интеграция системы сталкеров

## Описание
Данное руководство описывает интеграцию системы сталкеров с существующими компонентами игры: аномалиями, артефактами, ZoneController и системой навигации.

## Основные компоненты

### 1. Базовый класс сталкеров (`BaseStalker`)
- **Расположение**: `scripts/stalkers/base_stalker.gd`
- **Функциональность**:
  - Управление здоровьем, броней, скоростью
  - Система состояний (IDLE, PATROL, CHASE, ATTACK, FLEE, DEAD)
  - Навигация с использованием NavigationAgent3D
  - Интеграция с ZoneController
  - Сбор и использование артефактов

### 2. Типы сталкеров
- **NoviceStalker**: Начинающие сталкеры с пониженными характеристиками
- **VeteranStalker**: Опытные сталкеры с повышенными характеристиками

### 3. Аномалии (`AnomalyFire`)
- **Расположение**: `scripts/anomaly_fire.gd`
- **Функциональность**:
  - Нанесение урона сталкерам в зоне действия
  - Отслеживание входа/выхода сталкеров
  - Потребление энергии

### 4. Артефакты
- **Базовый класс**: `scripts/artifact.gd`
- **Типы**: CommonArtifact, EnergyArtifact, RareArtifact
- **Функциональность**:
  - Сбор сталкерами
  - Применение эффектов
  - Сигналы о сборе

### 5. ZoneController
- **Расположение**: `scripts/zone_controller.gd`
- **Функциональность**:
  - Управление ресурсами (энергия, биомасса)
  - Отслеживание сталкеров в зоне
  - Создание аномалий и артефактов

## Интеграция компонентов

### 1. Сталкеры и аномалии
Сталкеры правильно взаимодействуют с аномалиями:
- Аномалии определяют сталкеров по наличию методов `take_damage` и `is_alive`
- При входе в зону аномалии сталкер добавляется в список отслеживания
- Каждую секунду сталкер получает урон от аномалии
- При выходе из зоны сталкер удаляется из списка

### 2. Сталкеры и артефакты
Сталкеры могут собирать артефакты:
- Артефакты эмитируют сигнал `collected_by_stalker` при сборе
- Сталкеры имеют сигнал `artifact_collected` для обработки сбора
- Артефакты добавляются в инвентарь сталкера
- Применяются эффекты артефактов (восстановление здоровья, увеличение брони)

### 3. Сталкеры и ZoneController
Сталкеры интегрированы с ZoneController:
- Сталкеры автоматически подключаются к ZoneController при инициализации
- ZoneController отслеживает вход/выход сталкеров из зоны
- Сталкеры эмитируют сигналы `entered_zone` и `exited_zone`
- ZoneController управляет списком активных сталкеров

### 4. Система навигации
Система навигации работает корректно:
- NavigationAgent3D настроен для сталкеров
- Цели устанавливаются через `set_target_position`
- Движение к цели происходит через `get_next_path_position`
- Поддержка обхода препятствий

## Использование

### Создание сталкера
```gdscript
# Создание базового сталкера
var stalker = BaseStalker.new()
stalker.health = 100.0
stalker.speed = 5.0
stalker.damage = 10.0

# Добавление в сцену
add_child(stalker)
```

### Создание аномалии
```gdscript
# Создание огненной аномалии
var anomaly = AnomalyFire.new()
anomaly.damage_per_second = 15.0
anomaly.energy_cost_per_second = 3.0

# Добавление в сцену
add_child(anomaly)
```

### Создание артефакта
```gdscript
# Создание обычного артефакта
var artifact = CommonArtifact.new(Vector2(10, 10))
artifact.value = 15

# Добавление в сцену
add_child(artifact)
```

## Проблемы и решения

### 1. Несовместимость типов
- **Проблема**: Аномалии проверяли тип `Stalker`, но сталкеры наследуются от `BaseStalker`
- **Решение**: Проверка наличия методов `take_damage` и `is_alive`

### 2. Несовпадение сигналов
- **Проблема**: Артефакты эмитировали `collected_by_stalker`, а сталикеры ожидали `artifact_collected`
- **Решение**: Добавление дополнительной эмитации сигнала в методе `collect`

### 3. Отсутствие интеграции с ZoneController
- **Проблема**: Сталкеры не подключались к ZoneController
- **Решение**: Автоматическое подключение в методе `_ready`

### 4. Неполная реализация навигации
- **Проблема**: NavigationAgent3D не был настроен корректно
- **Решение**: Добавление полной настройки навигационных параметров
