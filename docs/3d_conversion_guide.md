# Руководство по конвертации проекта в 3D

## Обзор

Проект "Сталкер наоборот" был успешно конвертирован из 2D в 3D формат. Эта документация описывает основные изменения и улучшения, внесенные в процессе конвертации.

## Основные изменения

### 1. Скрипты

#### Сталкеры
- `BaseStalker`: конвертирован из `CharacterBody2D` в `CharacterBody3D`
- Добавлен `NavigationAgent3D` для навигации в 3D пространстве
- Обновлена физика движения с использованием `Vector3` вместо `Vector2`
- Добавлена 3D коллизия с `CapsuleShape3D`

#### Аномалии
- `AnomalyFire`: конвертирован из `Area2D` в `Area3D`
- `HeatAnomaly`: конвертирована из `Node2D` в `Area3D`
- `ElectricAnomaly`: конвертирована из `Node2D` в `Area3D`
- Обновлены методы для работы с `Vector3` вместо `Vector2`

#### Мутанты
- `BaseMutant`: конвертирован из `CharacterBody2D` в `CharacterBody3D`
- `DogMutant`: обновлен для работы с 3D
- `SnorkMutant`: обновлен для работы с 3D
- `ControllerMutant`: обновлен для работы с 3D

#### Артефакты
- `BaseArtifact`: конвертирован из `Node2D` в `Node3D`
- `CommonArtifact`: обновлен для работы с `Vector3`
- `RareArtifact`: обновлен для работы с `Vector3`
- `EnergyArtifact`: обновлен для работы с `Vector3`

#### Системы
- `ZoneController`: обновлен для работы с `Vector3`
- `StalkerSpawner`: обновлен для работы с `Vector3`
- `MainUI`: обновлен для 3D взаимодействия с камерой
- `ResourcePanel`: остался без изменений (UI элемент)

### 2. Сцены

#### Основные сцены
- `main.tscn`: корневая сцена конвертирована из `Node2D` в `Node3D`
- `test_3d_scene.tscn`: новая тестовая сцена с камерой для проверки 3D
- `zone_controller.tscn`: конвертирована из `Node2D` в `Node3D`
- `stalker_spawner.tscn`: конвертирована из `Node` в `Node3D`

#### Сцены сталкеров
- `base_stalker.tscn`: полностью обновлена для 3D с `CharacterBody3D`
- `novice_stalker.tscn`: обновлена для 3D
- `veteran_stalker.tscn`: обновлена для 3D

#### Сцены аномалий
- `fire_anomaly.tscn`: обновлена для 3D с `Area3D` и `MeshInstance3D`
- `heat_anomaly.tscn`: конвертирована из `Node2D` в `Area3D`
- `electric_anomaly.tscn`: конвертирована из `Node2D` в `Area3D`

#### Сцены мутантов
- `dog_mutant.tscn`: конвертирована из `CharacterBody2D` в `CharacterBody3D`
- `snork_mutant.tscn`: конвертирована из `CharacterBody2D` в `CharacterBody3D`
- `controller_mutant.tscn`: конвертирована из `CharacterBody2D` в `CharacterBody3D`

#### Сцены артефактов
- `base_artifact.tscn`: конвертирована из `Node2D` в `Node3D`

#### UI сцены
- `main_ui.tscn`: осталась без изменений (CanvasLayer работает в 3D)
- `resource_panel.tscn`: осталась без изменений

### 3. Физика и коллизии

#### 3D коллизии
- `CapsuleShape3D` для сталкеров и мутантов
- `SphereShape3D` для аномалий
- `BoxShape3D` для временных 3D моделей

#### Навигация
- `NavigationAgent3D` для сталкеров
- `NavigationRegion3D` для базового сталкера
- Обновлена система навигации для 3D пространства

### 4. Визуальные элементы

#### 3D модели
- `MeshInstance3D` вместо `Sprite2D`
- Временные кубы и сферы для визуализации
- `Label3D` для отображения имен сталкеров

#### Камера
- Добавлена `Camera3D` в тестовую сцену
- Настроена позиция и угол обзора

## Технические детали

### Векторы и позиции
- Все `Vector2` заменены на `Vector3`
- Y-компонента используется для высоты в 3D пространстве
- Для 2D объектов в 3D пространстве Y = 0

### Физика
- `move_and_slide()` для 3D движения
- `global_position` вместо `position` для 3D объектов
- `distance_to()` работает с `Vector3`

### UI взаимодействие
- 3D raycasting для определения позиции клика мыши
- Проекция 2D координат мыши в 3D пространство
- Использование `get_camera_3d()` для получения камеры

## Тестирование

### Тестовая сцена
- Создана `test_3d_scene.tscn` для проверки 3D функциональности
- Добавлен тестовый скрипт `test_3d_systems.gd`
- Настроена камера для обзора 3D сцены

### Тестовые проверки
- Проверка работы ZoneController
- Проверка создания аномалий
- Проверка создания мутантов
- Проверка UI элементов

## Настройки проекта

### project.godot
- Обновлена главная сцена на `test_3d_scene.tscn`
- Настроен 3D физический движок (Jolt Physics)
- Оптимизации для 3D рендеринга

### Заключение

Проект успешно конвертирован в 3D формат с сохранением всей функциональности. Все системы работают в 3D пространстве с использованием современных подходов Godot 4.6.1.
